## التوصيات النهائية لبناء واجهة “إقرأ 12” المخملية  
**Final UI Recommendations — Velvet Dashboard**

تمثل هذه الوثيقة خلاصة تحليل القدرات، والعمارة التشغيلية، ورحلة الباحث، والبروفايل الإدراكي، وعدسات التفكير، والوحدات، وعناصر الواجهة.  
وهي تُعدّ “دليل التنفيذ” لبناء لوحة القيادة النهائية.

---

# **1. المبادئ التأسيسية للواجهة**
### يعتمد تصميم الواجهة على ستة مبادئ عليا:

1. **العمق الإدراكي**  
   واجهة مظلمة مخملية ذات تدرجات عميقة.

2. **الهدوء البصري**  
   تقليل التداخلات والضوضاء البصرية.

3. **الطبقات المعمارية**  
   Layered UI بدلًا من الواجهات المسطحة.

4. **التسلسل الذهني**  
   احترام رحلة الباحث ومراحله الإدراكية.

5. **القدرات قبل المظهر**  
   كل زر يجب أن يعبر عن قدرة.

6. **التحكم الكامل للمستخدم**  
   الواجهة لا تفرض الإيقاع بل تتبعه.

---

# **2. بنية لوحة القيادة — Dashboard Architecture**
تتكون لوحة القيادة من:

### **أ. الشريط الجانبي الثابت — Persistent Sidebar**
يحتوي على:
- المفاتيح الأربع الأساسية  
- نظام العدسات  
- مفتاح المشاريع  
- مفتاح المسارات  

### **ب. منطقة العرض الرئيسية — Main Workspace**
تدعم:
- البطاقات  
- المسارات  
- الخرائط  
- العدسات  
- الوحدات  

### **ج. منطقة الأدوات الثانوية — Utility Zone**
تشمل:
- تصدير  
- أرشفة  
- مشاركة  
- وكلاء التنفيذ  

### **د. شريط العدسات — Lens Bar**
ينتقل الباحث من خلاله بين:
- العلاقات  
- التسلسل  
- التأثير  
- النطاق  
- البنية  
- التفسير  

---

# **3. نظام الألوان — Color System**
### النظام اللوني النهائي مستخرج من التحليل الإدراكي:

- الخلفية: كحلي مخملي (Deep Velvet Navy)  
- اللمسات: زيتي داكن (Deep Olive)  
- الإشارات الذهنية: ذهبي ناعم (Soft Gold)  
- الخلفيات الداخلية: طبقات رمادية باردة  
- التباينات: درجات فاتحة جدًا لضمان القراءة  

---

# **4. هيكلة عناصر الواجهة — Component Structure**
### تتكون العناصر من:

1. **المفاتيح الأساسية:**  
   Large Velvet Keys  
   تعمل كبوابات لقدرات المنصة.

2. **البطاقات:**  
   Concept Cards  
   تعرض النتائج والوحدات.

3. **الوحدات:**  
   Functional Modules  
   تحلل وتبني وتفهم.

4. **العدسات:**  
   Cognitive Lenses  
   تغير منظور الفهم.

5. **المسارات:**  
   Flow Tracks  
   تمثل مراحل تحليل أو بناء.

6. **الأزرار الوظيفية:**  
   Olive Rectangular Buttons  
   صغيرة ومستطيلة بحواف ناعمة.

---

# **5. توصيات الذكاء الاصطناعي للبناء — AI Implementation Guide**
ليتمكن جيمي من بناء الواجهة، يحتاج إلى:

### 1) قراءة جميع الملفات داخل capabilities/  
### 2) استخراج القدرات → مفاتيح  
### 3) استخراج العمارة → بنية لوحة القيادة  
### 4) استخراج الرحلة → تسلسل المستخدم  
### 5) استخراج العدسات → شريط العدسات  
### 6) استخراج البروفايل الإدراكي → نظام الألوان والطبقات  
### 7) ربط كل ذلك بمكونات React + Next.js
